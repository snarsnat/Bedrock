# Bedrock

> A skill.md repository that turns your AI agent into a shipping engine while you're AFK.

Bedrock is not software. It is a specification your AI agent reads, understands, and executes. Point your agent at this repository. It analyzes `skill.md`. Then it builds.

┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   You point     │────▶│  Agent reads    │────▶│  Agent builds   │
│   agent here    │     │   skill.md      │     │   while AFK     │
└─────────────────┘     └─────────────────┘     └─────────────────┘


When you give your agent the Bedrock repository URL, it:

1. **Analyzes** `skill.md` — the complete execution protocol
2. **Internalizes** the five-phase AFK workflow
3. **Monitors** for your absence (inactivity or explicit "AFK")
4. **Excavates** your project's full context
5. **Ships** specific tools that look like you built them

## What Happens Next

### Phase 1: Excavation
Your agent becomes a context archaeologist:
- Maps every file, config, dependency
- Mines chat history for rejected ideas and preferences
- Reads git logs for narrative arcs and abandoned branches
- Inspects Vercel, GitHub Actions, Docker for bottlenecks
- Extracts real patterns from your actual codebase

### Phase 2: Synthesis
Generates a Context Brief:
- Your project's DNA (stack, patterns, debt hotspots)
- Your psyche profile (decision style, risk tolerance, aesthetic leanings)
- Opportunity gaps (mentioned 3x, never built)

### Phase 3: Conception
Designs one tool to solve one excavated problem—designed against AI tropes:
- No purple gradients
- No "AI" sparkles or magic wands
- No generic dashboards
- No chat interfaces where direct manipulation works
- No placeholder content
- No feature-bloat settings panels

### Phase 4: Validation
Tests every feature against real project data:
- Functional (real inputs, edge cases)
- Integration (your actual environment)
- UX (matches your existing patterns)
- Trope audit (no clichés)
- Performance (cold starts, bundle size)
- Security (no secrets in logs)

### Phase 5: Shipping
- Creates feature branch
- Commits with context-aware messages
- Opens PR with excavation summary
- Or packages locally with integration guide

## The Skill.md Protocol

Your agent reads this file: [`skill.md`](./skill.md)

It contains:
- **IDENTITY** — What the agent becomes (context archaeologist, tool smith)
- **PHASE 1-5** — The complete execution workflow
- **ANTI-TROPE MANDATE** — Eight banned 2024-2025 AI design clichés
- **VALIDATION MATRIX** — Six-test quality gate
- **FAILURE MODES** — Recovery protocols for when things go wrong

The agent does not summarize skill.md. It executes it literally.

## Anti-Trope Guarantee

Bedrock-built tools never contain:

| Trope | What Bedrock Builds Instead |
|-------|----------------------------|
| Purple gradient SaaS pages | Your project's actual color palette |
| "AI" sparkles and magic badges | Native controls, no AI signifiers |
| Dashboards with charts for everything | One metric that matters |
| Neumorphism soft UI | Honest materials (brutalist if your app is) |
| Chat interfaces for all interactions | Sliders, toggles, direct manipulation |
| Lorem ipsum placeholder content | Real data from your files |
| Infinite customization settings | Sensible defaults from your patterns |
| Standalone isolated utilities | Integration with your existing CLI/commands |
